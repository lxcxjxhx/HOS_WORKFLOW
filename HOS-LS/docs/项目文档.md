# HOS-LS 安全检测工具 - 项目文档

## 1. 项目概述

HOS-LS 安全检测工具是一款专门针对各类AI开源小工具（通过各类AI IDE生成的工具）进行安全评估的工具。它能够检测代码安全、权限安全、网络安全、依赖安全和配置安全等多个方面的问题，并生成详细的安全评估报告。

随着公司AI化进程的不断深入，AI工具在教学和科研中的应用日益广泛。越来越多的老师提交的交付工具均由AI辅助生成，这些工具在提高效率的同时，也带来了新的安全挑战。由于AI生成的代码可能存在安全隐患，而开发过程中对安全性的关注相对不足，因此建立一套完善的安全检测机制变得尤为重要。

本系统作为保障AI工具安全开发的重要工具，将在以下方面发挥关键作用：
- **安全保障**：全面检测AI生成工具中的安全问题，确保代码的安全性和可靠性
- **合规性**：帮助老师提交的工具符合公司的安全标准和规范
- **风险预防**：提前发现并解决潜在的安全风险，避免安全事件的发生
- **持续改进**：通过定期的安全检测，促进AI工具开发质量的持续提升

通过本系统的安全检测，老师可以在提交交付工具时，确保代码的安全性和可靠性，为公司的AI化进程提供有力的安全保障。


### 1.1 项目目标

- 提供一个通用的安全检测工具，能够检测AI工具中的常见安全问题
- 生成详细的安全评估报告，帮助用户了解AI工具的安全状况
- 支持多种报告格式，满足不同用户的需求
- 提供友好的命令行界面，方便用户使用

### 1.2 技术栈

- **开发语言**：Python 3.8+
- **核心依赖**：
  - `pyyaml` - 配置文件解析
  - `markdown` - MD报告生成
  - `python-docx` - DOCX报告生成
  - `jinja2` - HTML模板渲染
  - `requests` - 网络请求（用于依赖安全检查）
  - `colorama` - 命令行彩色输出

## 3. 核心模块

### 3.1 安全扫描模块 (security_scanner.py)

安全扫描模块负责执行各种安全检测，包括：

- **代码安全检测**：检测硬编码的API密钥、密码、token等敏感信息，以及潜在的后门代码
- **权限安全检测**：检查文件的执行权限
- **网络安全检测**：检测硬编码的IP地址和端口号
- **依赖安全检测**：检查项目的依赖包
- **配置安全检测**：检测配置文件中的敏感信息

**新增功能**：
- **静默模式支持**：接收静默模式参数，根据参数控制输出行为
- **条件输出**：所有打印语句都添加静默模式检查
- **统一接口**：保持与原有功能的兼容性

**核心实现**：
```python
def __init__(self, silent=False):
    self.silent = silent

def scan_code_security(self, target):
    if not self.silent:
        print("正在检测代码安全...")
    # 检测逻辑...
```

### 3.2 报告生成模块 (report_generator.py)

报告生成模块负责根据扫描结果生成不同格式的报告：

- **HTML报告**：响应式设计，彩色风险等级标识
- **DOCX报告**：专业格式，包含目录和样式
- **MD报告**：简洁明了，支持GitHub渲染

### 3.3 主程序模块 (main.py)

主程序模块负责处理命令行参数，调用安全扫描模块执行扫描，然后调用报告生成模块生成报告。

**新增功能**：
- **静默模式**：通过 `-s/--silent` 参数启用，在执行过程中不显示控制台输出
- **参数解析**：添加对静默模式参数的支持
- **流程控制**：根据静默模式参数控制输出行为

**核心实现**：
```python
# 命令行参数解析
parser.add_argument('-s', '--silent', action='store_true', help='静默模式，不显示控制台输出')

# 安全扫描器初始化
scanner = SecurityScanner(silent=args.silent)

# 条件输出
if not self.silent:
    print("执行安全扫描...")
```

## 4. 安全检测范围

### 4.1 代码安全

- 硬编码的API密钥、密码、token等敏感信息
- 潜在的后门代码（如exec、eval、system等函数调用）
- 网络访问代码

### 4.2 权限安全

- 文件执行权限

### 4.3 网络安全

- 硬编码的IP地址
- 硬编码的端口号

### 4.4 依赖安全

- 项目依赖包

### 4.5 配置安全

- 配置文件中的敏感信息

## 5. 报告格式

### 5.1 HTML报告

HTML报告采用响应式设计，包含以下内容：
- 检测摘要：显示检测目标、检测时间、风险数量
- 详细的安全问题列表，按类别分组
- 彩色风险等级标识

### 5.2 DOCX报告

DOCX报告采用专业格式，包含以下内容：
- 标题和检测摘要
- 详细的安全问题列表，按类别分组
- 表格形式展示检测结果

### 5.3 MD报告

MD报告采用Markdown格式，包含以下内容：
- 检测摘要
- 详细的安全问题列表，按类别分组
- 表格形式展示检测结果

## 6. 部署与安装

### 6.1 环境要求

- Python 3.8+
- 操作系统：Windows、Linux、macOS

### 6.2 安装步骤

1. 下载或克隆项目到本地
2. 进入项目目录
3. 安装依赖包：
   ```bash
   pip install -r requirements.txt
   ```

### 6.3 运行方式

#### Windows系统

```bash
python src/main.py <目标路径> [选项]
```

#### Linux系统

```bash
chmod +x hos-ls.sh
./hos-ls.sh <目标路径> [选项]
```

### 6.4 Python打包

1. 生成wheel包：
   ```bash
   python setup.py bdist_wheel
   ```

2. 安装wheel包：
   ```bash
   pip install dist/hos_ls-1.0.0-py3-none-any.whl
   ```

3. 安装后运行：
   ```bash
   hos-ls <目标路径> [选项]
   ```

## 7. 开发与扩展

### 7.1 如何添加新的检测规则

1. 在 `security_scanner.py` 中添加新的检测函数
2. 在 `scan` 方法中调用新的检测函数
3. 更新报告生成模块，确保新的检测结果能够正确显示

### 7.2 如何添加新的报告格式

1. 在 `report_generator.py` 中添加新的报告生成方法
2. 在 `main.py` 中添加对新格式的支持
