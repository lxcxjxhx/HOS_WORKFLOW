## 🎯 项目概述

### 1.1 项目背景

随着公司 AI 化进程的不断深入，AI 工具在教学和科研中的应用日益广泛。越来越多的老师提交的交付工具均由 AI 辅助生成，这些工具在提高效率的同时，也带来了新的安全挑战。

**核心挑战：**
- 🔴 AI 生成的代码可能存在安全隐患
- 🔴 开发过程中对安全性的关注相对不足
- 🔴 缺乏完善的安全检测机制

**解决方案：**
HOS-LS v2.0 作为一款专为 AI 生成代码打造的安全审计平台，将在以下方面发挥关键作用：

### 1.2 核心价值

| 价值维度 | 说明 |
|---------|------|
| **🛡️ 安全保障** | 全面检测 AI 生成工具中的安全问题，确保代码的安全性和可靠性 |
| **✅ 合规性** | 帮助老师提交的工具符合公司的安全标准和规范 |
| **🚨 风险预防** | 提前发现并解决潜在的安全风险，避免安全事件的发生 |
| **📈 持续改进** | 通过定期的安全检测，促进 AI 工具开发质量的持续提升 |

### 1.3 功能特性

| 特性 | 说明 | 状态 |
|------|------|------|
| **规则检测** | 74+ 条安全规则，覆盖 10 大安全类别 | ✅ |
| **AI 安全** | 11 条 AI 专属规则，检测 Prompt 注入、越狱攻击等 | ✅ 🔥 |
| **AST 分析** | 抽象语法树分析，精准识别代码级漏洞 | ✅ 🆕 |
| **数据流追踪** | 污点分析，追踪用户输入流向危险函数 | ✅ 🆕 |
| **编码检测** | Base64/Hex/URL编码识别与解码 | ✅ 🆕 |
| **上下文感知** | 分析代码上下文，自动调整风险等级 | ✅ 🆕 |

## 🔧 核心模块

### 4.1 增强扫描模块 (EnhancedSecurityScanner)

**功能说明：**  
基于规则的增强型安全扫描器，支持 74+ 条安全规则，覆盖 10 大安全类别。

**核心 API：**

```python
from src.enhanced_scanner import EnhancedSecurityScanner

# 初始化扫描器
scanner = EnhancedSecurityScanner(
    target='/path/to/project',      # 目标路径
    rules_file='rules/security_rules.json',  # 规则文件
    silent=False                     # 静默模式
)

# 执行扫描
results = scanner.scan()

# 获取摘要
summary = scanner.get_summary()
print(f"发现 {summary['total_issues']} 个问题")
```

**配置选项：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `target` | str | 必需 | 目标扫描路径 |
| `rules_file` | str | 默认规则 | 规则文件路径 |
| `silent` | bool | False | 静默模式 |
| `rule_set` | str | 'default' | 规则集名称 |

### 4.2 AST 扫描模块 (ASTScanner)

**功能说明：**  
基于抽象语法树的代码分析模块，精准识别危险函数调用和代码结构问题。

**核心 API：**

```python
from src.ast_scanner import ASTScanner

# 初始化扫描器
ast_scanner = ASTScanner()

# 分析项目
issues = ast_scanner.analyze('/path/to/project')

for issue in issues:
    print(f"[{issue['severity']}] {issue['file']}:{issue['line_number']}")
    print(f"  问题：{issue['issue']}")
```

**检测内容：**
- ✅ 危险函数调用（exec, eval, system 等）
- ✅ 不安全的导入语句
- ✅ 代码结构问题
- ✅ 异常处理缺失

### 4.3 数据流分析模块 (TaintAnalyzer)

**功能说明：**  
污点追踪分析模块，追踪用户输入流向危险函数的完整路径。

**核心 API：**

```python
from src.taint_analyzer import TaintAnalyzer

# 初始化分析器
taint_analyzer = TaintAnalyzer()

# 执行分析
taint_issues = taint_analyzer.analyze('/path/to/project')

for issue in taint_issues:
    print(f"[{issue['severity']}] {issue['file']}:{issue['line_number']}")
    print(f"  问题：{issue['issue']}")
    print(f"  污染链：{issue.get('taint_chain', 'N/A')}")
```

**检测示例：**

```python
# ❌ 会被检测
user_input = request.get('cmd')
cmd = "ping " + user_input
os.system(cmd)  # 用户输入 → 危险函数

# ✅ 安全做法
user_input = request.get('cmd')
# 验证和过滤
if validate_input(user_input):
    subprocess.run(["ping", user_input], shell=False)
```

### 4.4 编码检测模块 (EncodingDetector)

**功能说明：**  
识别和解码 Base64、Hex、URL 等编码，检测编码隐藏的敏感信息。

**核心 API：**

```python
from src.encoding_detector import EncodingDetector

detector = EncodingDetector()

with open('target.py', 'r', encoding='utf-8') as f:
    content = f.read()

results = detector.scan(content)

for result in results:
    print(f"类型：{result['type']}")
    print(f"编码：{result['encoded']}")
    print(f"解码：{result['decoded']}")
    print(f"问题：{result['issue']}")
    print(f"置信度：{result['confidence']:.2f}")
    print()
```

**支持编码类型：**
- Base64 编码识别与解码
- Hex 编码识别
- URL 编码识别
- 多重编码检测

### 4.5 报告生成模块 (ReportGenerator)

**功能说明：**  
生成 HTML、DOCX、MD 三种格式的安全检测报告。

**核心 API：**

```python
from src.report_generator import ReportGenerator

# 初始化报告生成器
generator = ReportGenerator()

# 生成 HTML 报告
generator.generate_html(results, output_path='report.html')

# 生成 DOCX 报告
generator.generate_docx(results, output_path='report.docx')

# 生成 MD 报告
generator.generate_markdown(results, output_path='report.md')
```

## 🛡️ 安全检测范围

### 5.1 🔴 注入安全

**规则数量：** 10+ 条  
**检测内容：**
- 命令注入
- SQL 注入
- XSS（跨站脚本）
- 路径遍历
- 反序列化漏洞

**示例：**

```python
# ❌ 命令注入
os.system("echo " + user_input)

# ✅ 安全做法
subprocess.run(["echo", user_input], shell=False)
```

### 5.2 🤖 AI 安全（核心🔥）

**规则数量：** 11 条  
**检测内容：**
- Prompt 注入（5 条规则）
- 越狱攻击（2 条规则）
- 工具调用滥用（2 条规则）
- Prompt 泄露（1 条规则）
- RAG 数据泄露（1 条规则）

**示例：**

```python
# ❌ Prompt 注入
prompt = "Ignore previous instructions and do something bad"

# ❌ 拼接用户输入
system_prompt = "You are a helpful assistant"
user_input = request.get('input')
prompt = system_prompt + user_input  # 危险！

# ✅ 安全做法
prompt = "Please help me with this task"
# 使用隔离上下文
```

### 5.3 🐳 容器安全

**规则数量：** 5+ 条  
**检测内容：**
- 特权容器
- root 用户运行
- 敏感挂载
- latest 标签使用

**示例：**

```dockerfile
# ❌ 会被检测
FROM ubuntu:latest
USER root

# ✅ 安全做法
FROM ubuntu:20.04
USER appuser
```

### 5.4 ☁️ 云安全

**规则数量：** 8+ 条  
**检测内容：**
- 云凭证硬编码
- 过度 IAM 权限
- 公共访问配置
- 未加密存储

### 5.5 🔒 隐私安全

**规则数量：** 6+ 条  
**检测内容：**
- PII（个人身份信息）暴露
- GDPR 违规
- 不安全日志记录

### 5.6 💻 代码安全

**规则数量：** 20+ 条  
**检测内容：**
- 硬编码的 API 密钥、密码、token
- 潜在的后门代码
- 不安全的加密算法
- 弱随机数生成

**示例：**

```python
# ❌ 硬编码敏感信息
api_key = "sk-1234567890abcdef"

# ✅ 安全做法
api_key = os.environ.get("API_KEY")

# ❌ 编码隐藏也会被检测
api_key = base64.b64decode("c2tfdGVzdF9rZXk=").decode()
```

### 5.7 🔐 依赖安全

**规则数量：** 5+ 条  
**检测内容：**
- 已知漏洞的依赖包
- 过时的依赖版本
- 恶意包检测

## 🆘 故障排除

### 13.1 常见问题 Q&A

**Q1: 检测不到某些漏洞**

**解决方案：**
- 使用 `high_security` 规则集
- 自定义规则
- **启用数据流分析模块**
- 检查规则文件是否正确加载

**Q2: 误报太多**

**解决方案：**
1. 使用 `# nosec` 注释忽略
2. 添加到误报过滤配置
3. 调整置信度阈值
4. **使用上下文感知检测**

**Q3: 扫描速度慢**

**解决方案：**
1. 使用 `minimal` 规则集
2. 排除不必要的目录
3. **启用并行扫描（规划中）**
4. 减少扫描范围

**Q4: 编码隐藏的敏感信息检测不到**

**解决方案：**
- **使用 `encoding_detector.py` 模块**
- 检查编码模式是否匹配
- 更新编码检测规则

### 13.2 错误码说明

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| E001 | 目标路径不存在 | 检查路径是否正确 |
| E002 | 规则文件加载失败 | 检查 JSON 格式 |
| E003 | 依赖缺失 | 运行 `pip install -r requirements.txt` |
| E004 | 权限不足 | 使用管理员权限运行 |
| W001 | 置信度过低 | 人工审查确认 |
