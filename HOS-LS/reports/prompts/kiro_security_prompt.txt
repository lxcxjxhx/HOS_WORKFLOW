## 敏感信息处理
要求：禁止在代码中硬编码密码、API密钥等敏感信息，应使用环境变量或安全配置管理。
示例：```python
# 错误做法
db_password = "MyS3cr3tP@ssw0rd"

# 正确做法
import os
db_password = os.environ.get('DB_PASSWORD')
```

## 危险函数调用
要求：避免使用可能导致命令注入或代码执行的危险函数（如`eval`, `os.system`），优先使用安全的替代方案。
示例：```python
# 错误做法
user_input = input("Enter calculation: ")
result = eval(user_input)  # 危险！

# 正确做法
# 使用安全的数学库或严格限制的解析器
```

## 输入验证与过滤
要求：对所有外部输入（用户输入、API响应、文件内容）进行严格的验证、过滤和转义，防止注入攻击。
示例：```python
# 错误做法
query = f"SELECT * FROM users WHERE name = '{user_input}'"  # SQL注入风险

# 正确做法
import sqlite3
conn = sqlite3.connect('database.db')
cursor = conn.cursor()
cursor.execute("SELECT * FROM users WHERE name = ?", (user_input,))  # 参数化查询
```

## 文件权限管理
要求：为创建的文件和目录设置最小必要权限，避免使用过于宽松的权限（如`777`）。
示例：```python
# 错误做法
with open('config.json', 'w') as f:
    f.write(data)
os.chmod('config.json', 0o777)  # 权限过高

# 正确做法
import os
with open('config.json', 'w') as f:
    f.write(data)
os.chmod('config.json', 0o600)  # 仅所有者可读写
```

## 网络请求安全
要求：发起网络请求时使用HTTPS协议，并验证SSL证书有效性，避免中间人攻击。
示例：```python
# 错误做法
import urllib.request
response = urllib.request.urlopen('http://example.com/data')  # 未加密

# 正确做法
import urllib.request
import ssl
context = ssl.create_default_context()
response = urllib.request.urlopen('https://example.com/data', context=context)
```
